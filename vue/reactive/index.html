# 什么MVC？

MVC常用于之前前后端不分离的项目。

  • M: model数据库中的数据
  • V: view前端页面
  • C: controller后端的控制器，用于做业务逻辑处理

# 谈谈对MVVM的理解？

MVVM常用于前后端分享的项目。由Medel、View、ViewModel组成

  • M：后端数据库中的数据
  • V: 前端UI页面或组件
  • VM: 同步View和Model。比如Vue中数据的响应式
